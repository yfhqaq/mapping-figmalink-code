# 评估模块方案（正确性与收益）

本文档描述如何评估“Figma 组件指纹映射到前端组件”的有效性与正确性，重点用于：
- 发现“未录入 UX 组件”导致的匹配失败
- 校验“已录入映射是否正确”
- 量化映射是否提升了 LLM 生成的准确率

## 1. 评估目标
1) 判断匹配是否正确  
2) 识别缺失组件/错误映射  
3) 量化映射对生成质量的提升  
4) 给出可调整的改进建议  

## 2. 评估输入
- 业务组件或页面链接（Figma）
- 映射上下文（`mapping-llm.json` + `mapping-llm.dictionary.json`）
- 可选：业务代码片段或已有实现

## 3. 评估输出（核心交付）
### 3.1 匹配报告（可人工复核）
报告必须包含：
- 业务组件/页面链接  
- 匹配到的 UX 组件链接（可跳转）  
- 匹配置信度  
- 触发规则（指纹/复合规则/回退规则）  
- 是否存在多候选（Top N 候选）  

### 3.2 失败报告
必须区分失败类型：
- **缺失 UX 组件**（上下文里没有该指纹）  
- **映射缺失**（指纹存在，但没有对应映射规则）  
- **复合规则不完整**（组合结构识别不全）  
- **候选冲突**（多个 UX 组件均匹配，难以决定）  

## 4. 核心评估维度
### 4.1 匹配正确性
**目的**：验证业务组件是否匹配到正确 UX 组件。  
**方法**：输出“业务组件链接 + UX 组件链接”，人工比对确认。  
**指标**：  
- 命中率 = 正确匹配数 / 总匹配数  
- Top N 命中率（如 Top 3）  

### 4.2 缺失覆盖
**目的**：发现“未录入 UX 组件”或“映射缺失”。  
**指标**：  
- 缺失指纹数量  
- 缺失映射数量  

### 4.3 复合规则正确性
**目的**：验证“多组件 → 单组件 props/slots”的合理性。  
**方法**：检查组合结构是否正确映射到目标组件。  
**指标**：  
- 复合命中率  
- 复合漏判率  

### 4.4 生成结果增益
**目的**：验证启用映射后生成结果是否更接近预期。  
**方法**：  
- 同一业务页面，分别生成“映射前”与“映射后”结果  
- 对比组件选择准确率、props 还原度  
**指标**：  
- 组件准确率提升  
- 关键 props 还原率提升  

## 5. 必要评估场景（建议）
1) **缺失组件场景**  
   - 业务页面存在 UX 组件，但映射库中缺失  
   - 评估输出是否提示缺失并给出候选  
2) **错误映射场景**  
   - 映射规则不准确导致匹配错  
   - 评估报告是否可定位错误规则  
3) **复合结构场景**  
   - 表格、搜索区、工具栏组合  
   - 评估是否能完整映射到 Table props  
4) **多候选场景**  
   - 多个 UX 组件可实现同一业务组件  
   - 评估是否输出 Top N 候选并可人工裁定  

## 6. 上下文优化点（可改进的方向）
以下是通过改善上下文就能提升正确性的点：
- **补全 UX 组件指纹**（减少缺失命中）  
- **完善复合规则**（多组件 → 单组件 props/slots）  
- **加场景约束**（如 scope = Table.columns / Form.items）  
- **多候选映射**（允许一个业务组件映射多个 UX 组件候选）  
- **props 语义补充**（例如 label/状态/variant 的解释）  

## 7. 报告格式建议（简化版）
```
业务组件链接: <url>
匹配结果:
  - UX组件: <url> (置信度: 0.82, 来源: componentId)
  - UX组件: <url> (置信度: 0.61, 来源: 结构回退)
问题/备注:
  - 该组件库中未发现 "xxx" 组件，建议补录
```

## 8. 最小侵入接入方式
- 评估模块独立运行，不影响主流程  
- 产出报告供人工复核  
- 通过调整映射文件逐步优化

## 10. 解析深度与常见坑
1) **层级过浅**  
   - 页面包含多层嵌套组件，浅层抓不到子组件指纹。  
   - 解决：页面场景建议 depth=4 或更深；组件场景 depth=2。
2) **实例不来自组件库**  
   - 业务页面使用了复制粘贴的图层而非组件实例。  
   - 解决：报告提示“无指纹”，推动 UX 使用组件实例。
3) **同名不同组件**  
   - 名称相似但指纹不同，容易误判。  
   - 解决：必须依赖 componentId/componentSetId，而非 name。
4) **多候选冲突**  
   - 多个 UX 组件都能实现业务组件。  
   - 解决：输出 Top N + 置信度 + 触发规则，人工裁定。
5) **复合结构漏判**  
   - 表格/表单等组合结构被拆散识别。  
   - 解决：补全复合规则与 scope，必要时加场景约束。
